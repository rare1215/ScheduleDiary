### 1. Business purpose
&nbsp; 최근 Y2K 열풍으로 인해 이전에 유행했던 다이어리 꾸미기가 다시 알음알음 부상하고 있다. 이는 주로 젊은 여성층을 중심으로 종이에 스티커나 마스킹 테이프 등을 붙이며 꾸미고 SNS에 올려 공유하는 것으로 자주 볼 수 있다. <br>
&nbsp; 이때 종이에 직접 오리고 붙여 꾸미는 아날로그의 매력도 있지만 이를 디지털로 할 수 있으면 어떨까하는 생각이 들었다. 그 이유는 첫째, 다이어리를 꾸미기 위한 용품들의 자리 차지가 크다. 스티커는 물론이고 마스킹 테이프 등 기본적으로 여러 장의 인쇄물로 이루어져 있기에 부피 차지가 꽤 크다. 둘째, 관리가 힘들다. 앞서 말했듯 양이 많기에 내가 원하는 용품을 찾으려면 오랜 시간이 걸린다. 셋째, 자원 낭비. 다이어리를 꾸밀 때 쓰는 메모지 등은 대량으로 파는 경우가 대다수라 조금 쓰고 쓰지 않게 되어 결국 버리게 된다. 이는 자원과 돈 둘 다 낭비된다고 볼 수 있다. <br>
&nbsp; 기존에도 수많은 다이어리 어플리케이션이 존재한다. 하지만 그들은 꾸밈의 자유도가 너무 낮고, 이미지의 첨부도 어렵다. 그래서 조금 더 자유도를 높인 프로그램을 만들어 다이어리를 쓰고 꾸밀 수 있도록 하면 사용자들이 만족감을 느낄 수 있을 것이라 예측한다. <br><br>

### 4. Concept of operation
1) 회원가입

|||
| :--- | :--- |
| Purpose | 사용자, 관리자가 시스템에 등록되어 있는지 확인 후 시스템을 이용할 수 있어야 한다. |
| Approach | 생성된 아이디와 비밀번호가 없을 시에 회원가입을 할 수 있게 한다. |
| Dynamics | 어플리케이션을 최초 실행해 계정이 없는 경우 |
| Goals | 데이터베이스에 등록되지 않은 사용자를 데이터베이스에 등록해, 시스템을 이용할 수 있게 한다. |

2) 로그인

|||
| :--- | :--- |
| Purpose | 사용자마다 다이어리 페이지가 분리되기에 개별 사용자를 구분할 수 있어야 한다. |
| Approach | 어플리케이션 실행 시에 아이디와 비밀번호를 받아 로그인을 할 수 있게 한다.  |
| Dynamics | 앱을 실행했을 경우 |
| Goals | 로그인을 완료한 사용자들이 권한에 따라 시스템을 이용할 수 있게 한다. |

3) 로그아웃

|||
| :--- | :--- |
| Purpose | 다른 아이디로 로그인 하거나, 다른 사람이 내 계정을 보는 것을 막기 위해 로그아웃을 할 수 있어야 한다. |
| Approach | 로그아웃 버튼을 만들어 로그아웃 할 수 있게 한다. |
| Dynamics | 로그아웃을 하고싶을 경우 |
| Goals | 로그아웃을 할 수 있게 한다. |

4) 새 다이어리 생성

|||
| :--- | :--- |
| Purpose | 다이어리 페이지를 만들기 전에 먼저 이를 묶을 다이어리가 필요하다. |
| Approach | 새 다이어리 생성 버튼을 누르면 새로운 다이어리를 생성한다. |
| Dynamics | 새 다이어리를 만들고 싶을 경우 |
| Goals | 새 다이어리를 생성한다. |

5) 새 다이어리 페이지 생성

|||
| :--- | :--- |
| Purpose | 다이어리를 꾸미기 위해 새 페이지를 만들어야 한다. |
| Approach | 새 페이지 생성 버튼을 만들어 새 페이지를 만든다. 생성위치는 이전 페이지의 뒤로 할 수도, 원하는 위치로 지정할 수도 있다. |
| Dynamics | 새 페이지를 만들고 싶을 경우 |
| Goals | 새 페이지를 생성한다. |

6) 이전 다이어리 조회

|||
| :--- | :--- |
| Purpose | 이전에 작업한 다이어리와 그 페이지를 볼 수 있어야 한다. |
| Approach | 이전에 작업한 다이어리의 리스트를 보여주고 클릭하면 해당 페이지로 이동할 수 있다. |
| Dynamics | 이전 다이어리를 보고싶을 경우 |
| Goals | 뭔하는 이전 다이어리를 보여준다. |

7) 배경 편집

|||
| :--- | :--- |
| Purpose | 사용자의 편리성을 위해 위에 올라가는 요소들과 별개로 배경을 편집하고 관리할 수 있어야 한다. |
| Approach | 배경 편집 버튼을 클릭하면 배경 이미지를 자유롭게 삽입할 수 있다. |
| Dynamics | 배경을 편집하고 싶을 경우 |
| Goals | 배경을 자유롭게 편집할 수 있게 한다. |

8) 텍스트 입력

|||
| :--- | :--- |
| Purpose | 사용자가 원하는 위치에 텍스트를 입력할 수 있어야 한다. |
| Approach | 텍스트 생성을 선택하고 페이지에서 원하는 위치를 누르면 해당 위치에 빈 텍스트 박스가 생성된다. 사용자는 이 텍스트 박스에 원하는 텍스트를 입력할 수 있고, 빈 텍스트 박스는 자동으로 삭제된다. 별도의 ui로 크기, 색, 두께, 폰트 등을 조정할 수 있다. |
| Dynamics | 페이지에 텍스트를 입력하는 경우 |
| Goals | 페이지에 텍스트박스를 만들어 입력할 수 있게 한다. |

9) 오브젝트 삽입

|||
| :--- | :--- |
| Purpose | 사용자가 원하는 위치에 오브젝트를 삽입할 수 있어야 한다. |
| Approach | 오브젝트 삽입을 선택하고 페이지에서 원하는 위치를 누르면 해당 위치에 새 오브젝트가 생성된다. 드래그로 크기와 위치를 조정할 수 있다. |
| Dynamics | 페이지에 오브젝트를 삽입하는 경우 |
| Goals | 페이지에 오브젝트를 생성해 자유롭게 편집할 수 있게 한다. |

10) 이미지 삽입

|||
| :--- | :--- |
| Purpose | 사용자가 원하는 위치에 이미지를 삽입할 수 있어야 한다. |
| Approach | 이비지 삽입을 누르면 사용자의 파일로 연결되고, 파일 탭에서 이미지를 선택하면 해당 이미지가 페이지에 나타난다. 드래그로 크기와 위치를 조정할 수 있다. |
| Dynamics | 페이지에 이미지를 삽입하는 경우 |
| Goals | 페이지에 원하는 이미지를 삽입해 자유롭게 편집할 수 있게 한다. |

11) 요소 정렬

|||
| :--- | :--- |
| Purpose | 사용자가 원하는 요소들을 선택해 정렬할 수 있어야 한다. |
| Approach | 사용자가 요소 정렬을 누르고 일정 요소들을 선택하면 자동 정렬된다. 옵션으로는 중앙정렬, 좌측정렬, 우측 정렬이 있다. |
| Dynamics | 특정 요소들을 일정한 규칙에 따라 정렬하는 경우 |
| Goals | 선택한 요소들을 규칙(가운데, 좌측, 우측)에 따라 정렬한다. |

12) 뒤로 돌리기

|||
| :--- | :--- |
| Purpose | 사용자가 원하지 않는 작업일 때 그를 뒤로 돌릴 수 있어야 한다. |
| Approach | 뒤로 돌리기 버튼을 누르면 직전에 완료했던 작업으로 돌아간다. |
| Dynamics | 이전의 작업을 실행 취소하는 경우 |
| Goals | 작업을 실행취소한다. |

13) 다시 실행

|||
| :--- | :--- |
| Purpose | 사용자가 취소한 작업을 다시 되돌리고 싶을 때 복구할 수 있어야 한다. |
| Approach | 다시 실행 버튼을 누르면 직전에 실행 취소했던 작업을 다시 실행한다. |
| Dynamics | 실행 취소한 작업을 다시 복구하는 경우 |
| Goals | 취소한 작업을 다시 실행한다. |

14) 테두리 넣기

|||
| :--- | :--- |
| Purpose | 사용자가 원하는 요소에 테두리를 넣고 싶을 때 넣을 수 있어야 한다. |
| Approach | 요소를 선택하고 테두리 넣기 버튼을 누르면 요소의 바깥에 테두리가 생성된다. 별도의 ui로 두께 조절이 가능하다. |
| Dynamics | 페이지에 삽입한 요소에 테두리를 넣고싶은 경우 |
| Goals | 선택한 요소에 테두리를 넣는다. |

15) 저장하기

|||
| :--- | :--- |
| Purpose | 사용자가 원할 때 작업을 저장할 수 있어야 한다. |
| Approach | 저장 버튼을 만들어 원하는 시점에 저장할 수 있게 한다. |
| Dynamics | 페이지를 저장하는 경우 |
| Goals | 페이지를 저장한다. |

### 5. Problem statemen

Overview
Schedule Diary는 사용자가 일기를 쉽게 쓰고 꾸밀 수 있게 하여 다이어리 기록의 점근성을 높이는 프로그램이다. 또한 개개인의 일상을 기록해 후에 추억으로 보존할 수도 있게 한다.
위의 목적을 달성하기 위해선 아래와 같은 유의사항이 존재한다.

* Problem#1
&nbsp; Schedule Diary는 사용자가 매번 요소를 추가하고 삭제할 수 있고 이가 계속 쌓여가는 구조이다. 그에 따라 많을 용량과 데이터베이스를 차지하기에 앱이 무거워지거나, 사용 시 버퍼링이 걸릴 수 있다. 이런 사용자의 불편함을 방지하기 위해 이전 작업에 대한 데이터베이스의 효율적인 관리와 서버의 최적화가 필요하다.
* Problem#2
&nbsp;  Schedule Diary는 개개인의 개인정보가 담긴 일기장 어플리케이션이다. 그렇기에 Schedule Diary에 작성된 내용들은 개인과 관리자만 접속할 수 있어야 하며 외부 인원이 접근할 시 크나큰 문제가 발생하게 된다. 그래서 사용자 개개인의 데이터 분리를 철저히 하고, 외부인이 쉽게 접근할 수 없게 보안을 철저히 해야한다.
* Problem#3
&nbsp;  Schedule Diary는 모바일 구동을 전제로 한 어플리케이션이다. 하지만 이미지의 편집이라는 작업이 무겁기에 모바일에서는 앱이 잘 작동하지 않을 수도 있다. 그리고 pc를 주로 사용하는 사용자는 앱에 대한 접근성이 떨어질 수 있다.
* Problem#4
&nbsp;  Schedule Diary는 이미지를 편집하여 저장하는 프로그램이라 저장소의 효휼적이고 정확한 관리가 필요하다. 이미지를 최적화 없이 불러오게 된다면 앱이 가동을 정지할 수 있다. 또한 많은 이미지가 있는 페이지를 열 때도 앱이 멈추게 된다. 그리고 저장소가 꼬이게 된다면 엉뚱한 다이어리에 페이지가 가 있는 등 큰 불편을 야기할 수 있다.
